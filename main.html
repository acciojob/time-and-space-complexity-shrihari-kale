Time and Space Complexity

Time complexity measures how the running time of an algorithm grows relative to the size of its input. It is expressed using big-O notation (for example O(1), O(n), O(n log n)) to capture the dominant cost as input scales. When designing algorithms we often aim for solutions that are more efficient in time so they complete faster on large inputs.

Space complexity describes the amount of memory an algorithm requires as a function of input size. Space Complexity accounts for temporary structures, recursion stacks, and any extra storage beyond the input itself. For example, an algorithm that needs an extra array proportional to the input has space complexity O(n), while an in-place algorithm that only uses a few variables is O(1).
